Hamacher §1.1 · One design, every size

Four kinds of computer, same five boxes

Embedded

Hidden inside a device, one fixed job — washing machine, car, water heater. You never see it.

Personal

Desktop, workstation, laptop — general work for one person.

Servers & Enterprise

Shared over a network by many users — databases, banking, IRCTC.

Supercomputers & Grid

Highest performance: weather forecasting, simulation. Grids share the load across many machines.

Cloud computing = renting server time as a utility, pay-as-you-use. Inside every one of these: the same five units.

The basic functional units

Interconnection network — how the units exchange information (Unit 0 called them buses).

Input — accepts coded information from the outside.

Memory — stores programs AND data.

ALU — performs the processing steps.

Output — sends results back to the world.

Control — coordinates ALL of these actions.

Two teams: Input+Output = I/O; ALU+Control = the processor.

Everything inside is instructions or data

Machine instructions

Explicit commands that ① move information within the computer and to/from I/O devices, ② specify the arithmetic / logic operations to perform. A program = a list of instructions.

Data

Numbers and characters used as operands by the instructions.

Both are stored in memory, both encoded the same way: strings of bits — remember Unit 0.2, everything is switches.

Unit 1 of 5 · the senses

Input unit — the machine's senses

Press a key → the letter is instantly translated to its binary code and sent to the processor. The keyboard is the classic input device.

Also: touchpad, mouse, joystick, trackball · microphone (sound sampled into digital codes) · camera (video).

And other computers — the Internet is an input device too.

Unit 2 of 5 · Primary memory

Memory — cells, words, addresses

Millions of storage cells, each holding one bit — handled in fixed-size groups called words.

Word length = bits per word: typically 16, 32 or 64. One word is stored or retrieved in one basic operation.

Each word location has a distinct address: consecutive numbers 0, 1, 2, … (the lockers of Unit 0.4).

RAM: ANY location, short and fixed access time — a few ns to ~100 ns — regardless of position.

🙋

A computer has word length 32 bits.
How many bytes per word — and can you name a 64-bit CPU you own?

32 ÷ 8 = 4 bytes. And almost every phone in this room runs a 64-bit processor — 8-byte words.
Still unit 2 of 5 · the desk and the library

Cache — memory's fast little helper

Analogy: books you're using now stay on your desk, not on the library shelf. Fetching from the desk is instant.

The cache is a smaller, faster RAM on the processor chip itself, holding the program sections and data in current use.

Why it wins: program loops reuse the same instructions again and again — after the first pass, they're already in the cache.

Remember this idea — it returns as the star of Unit 3 (Pipelining) and Unit 4 (Memory).

Still unit 2 of 5

Secondary storage — big, cheap, permanent

Primary memory's two weaknesses: expensive, and it forgets when power goes off.

Secondary storage — magnetic disks, optical disks (DVD/CD), flash — is cheap and permanent, but slower.

Unit 3 of 5 · the calculator

Arithmetic & Logic Unit

Every arithmetic or logic operation — add, subtract, multiply, divide, compare — happens here. Operands are brought INTO the processor first.

They wait in registers — high-speed storage, one word each — faster even than the cache.

And inside the ALU's adder? The XOR + AND gates you built in Unit 0.2. It's gates all the way down.

🙋

Rank by speed, fastest first:
main memory · registers · cache

Registers → cache → main memory. The closer to the ALU, the faster (and smaller, and costlier). This staircase becomes the memory hierarchy of Unit 4.
Unit 4 of 5

Output unit — results to the world

The input unit's counterpart: printers (laser, inkjet — 20+ pages/min, yet mechanical, so slow beside the processor), displays, speakers, network.

Some devices do both: a touchscreen displays (output) and senses your finger (input) — hence the combined name I/O unit.

Unit 5 of 5 · the nerve centre

Control unit — the conductor

Memory, ALU and I/O each do their own work — someone must coordinate them. The control unit sends control signals to the others and senses their states.

Its main tool: timing signals — they decide WHEN each transfer happens, like a conductor's baton.

Surprise: it's not one neat box. Control circuitry is physically distributed throughout the machine, linked by control lines.

Hamacher's summary — learn these four lines

The operation of a computer

① The computer accepts programs and data through the input unit, into memory.

② Information is fetched from memory under program control into the ALU, and processed.

③ Processed results leave through the output unit.

④ ALL activities are directed by the control unit.
